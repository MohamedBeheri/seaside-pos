# 🌊 seaside — نظام نقاط البيع وإدارة المخازن

نظام متكامل (POS + ERP مصغّر) لمطعم/كافيه، بتصميم فاتح وبسيط، **كل شيء فيه ديناميكي** (الأصناف، التصنيفات، المكونات، المخازن، الموردون، طرق الدفع، الطاولات، الضريبة، الإعدادات).

**واجهة ثنائية اللغة (عربي/إنجليزي)** بزر تبديل يقلب الاتجاه RTL/LTR، و**ثيم فاتح/داكن** قابل للتبديل ومحفوظ. الفاتورة المطبوعة **بالإنجليزية** بلوجو seaside و**الدفع نقدي فقط**.

## المميزات

| الوحدة | الوصف |
|---|---|
| 🧾 **نقطة البيع** | شبكة أصناف مرئية + بحث + تصنيفات، سلة بطلب صالة/سفري/توصيل، طاولة وعدد أفراد ونادل، خصم وبقشيش، نافذة دفع متعددة (نقدي/فيزا/محفظة/QR) مع حساب الباقي وطباعة فاتورة حرارية. |
| 👨‍🍳 **شاشة المطبخ (KDS)** | الطلبات الجارية، تتبّع حالة كل صنف (جديد ← تحضير ← جاهز ← تم التقديم). |
| 🍽️ **الأصناف والوصفات (BOM)** | ربط كل صنف بمكوناته الخام والكميات بالوحدة الصغرى، وحساب التكلفة وهامش الربح تلقائياً. |
| 📦 **المخزون** | أرصدة المواد الخام، قيمة المخزون، تنبيهات حد إعادة الطلب، سجل حركة المخزن. |
| 🚚 **المشتريات** | استلام فواتير الموردين مع تحديث **متوسط التكلفة المتحرك** (Moving Average) وإعادة حساب تكلفة الأصناف المرتبطة. |
| 🗑️ **التوالف والهدر** | تسجيل المواد التالفة وفصل تكلفتها عن المبيعات. |
| 🔍 **الجرد (Variance)** | مطابقة الرصيد الدفتري بالفعلي واعتماد فروقات العجز/الفائض كتسويات. |
| 📊 **لوحة المعلومات** | مؤشرات لحظية (طلبات/مبيعات/أرباح اليوم)، منحنى المبيعات، أكثر الأصناف مبيعاً، طرق الدفع. |
| 📈 **تقارير الحوكمة** | الربحية وتكلفة المكونات، تقرير الهدر والعجز، أداء الموردين. |
| 👥 **الموظفون والصلاحيات** | أدوار (مدير/مدير فرع/كاشير/نادل/شيف) بصلاحيات مختلفة على الشاشات. |

## الحوكمة (منع الهدر والتلاعب)
- **الخصم التلقائي (Back-flush):** بمجرد تأكيد الدفع تُخصم مكونات الوصفة من المخزن الفرعي المعني، وتُسجَّل حركة مخزن لكل مادة.
- **الأسعار والتكلفة من الخادم:** لا تُحتسب من الواجهة لمنع التلاعب.
- **سجل تدقيق** لكل العمليات الحساسة.

## التشغيل محلياً
```bash
npm install
npm start          # http://localhost:4700
```
> يتطلب **Node 22.5+** (يستخدم `node:sqlite` المدمج — لا توجد تبعيات قاعدة بيانات خارجية).

قاعدة البيانات تُنشأ وتُبذَر تلقائياً عند أول تشغيل. لإعادة التهيئة من الصفر:
```bash
npm run reset && npm start
```

## النشر على Render
ادفع المستودع إلى GitHub ثم: **New + → Blueprint → اختر الريبو** (يقرأ `render.yaml` تلقائياً).
> الخطة المجانية تُعيد ضبط القرص دورياً؛ لحفظ البيانات بشكل دائم فعّل قرصاً (Disk) كما هو موضّح في `render.yaml`.

## البنية
```
server.js              ← REST API (Express) + منطق الخصم والمتوسط المتحرك والجرد
db/schema.sql          ← مخطط قاعدة البيانات الكامل
db/seed.js             ← البيانات الأولية (أصناف/مكونات/وصفات)
db/database.js         ← طبقة الاتصال (node:sqlite)
public/index.html      ← الواجهة
public/app.js          ← تطبيق الصفحة الواحدة (SPA)
public/styles.css      ← ثيم «البحر» الفاتح
```
